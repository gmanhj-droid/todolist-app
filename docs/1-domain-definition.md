## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| 프로젝트명 | TodoList |
| 목적 | 개인별 할일을 카테고리와 종료일 기준으로 체계적으로 관리하는 애플리케이션 |
| 핵심 문제 | 카테고리 분류 및 종료일 기반 상태 구분을 지원하는 개인 할일 관리 도구 부재 |
| 대상 사용자 | 할일을 체계적으로 관리하고자 하는 개인 사용자 |

---

## 2. 핵심 도메인 용어 (Ubiquitous Language)

| 용어 | 정의 |
|---|---|
| **사용자 (User)** | 회원가입 후 인증을 통해 앱을 사용하는 개인 |
| **할일 (Todo)** | 사용자가 등록한 처리해야 할 작업 단위 |
| **카테고리 (Category)** | 사용자가 직접 정의하는 할일의 분류 그룹 |
| **종료일 (Due Date)** | 할일을 완료해야 하는 목표 날짜 |
| **완료 (Completed)** | 사용자가 할일을 처리 완료로 표시한 상태 |
| **기한 초과 (Overdue)** | 종료일이 현재 날짜보다 이전이며 미완료인 상태 |
| **활성 (Active)** | 종료일이 미도래하였으며 미완료인 상태 |

---

## 3. 도메인 모델

### 3.1 핵심 엔티티 및 속성

**User (사용자)**
- `id` : 고유 식별자
- `email` : 로그인 이메일 (고유값, 이메일 형식)
- `password` : 암호화된 비밀번호 (최소 8자 이상)
- `created_at` : 가입일시

**Category (카테고리)**
- `id` : 고유 식별자
- `user_id` : 소유 사용자 (User 참조)
- `name` : 카테고리 이름 (최대 20자)
- `created_at` : 생성일시

**Todo (할일)**
- `id` : 고유 식별자
- `user_id` : 소유 사용자 (User 참조)
- `category_id` : 분류 카테고리 (Category 참조, 선택)
- `title` : 할일 제목 (최대 50자)
- `description` : 상세 내용 (최대 200자, 선택)
- `due_date` : 종료일 (선택)
- `is_completed` : 완료 여부
- `created_at` : 생성일시
- `updated_at` : 수정일시

### 3.2 엔티티 관계

```
User 1 ──< Category (사용자가 카테고리를 소유)
User 1 ──< Todo     (사용자가 할일을 소유)
Category 1 ──< Todo (카테고리가 할일을 분류, 선택적)
```

## 4. 핵심 비즈니스 규칙

| # | 규칙 |
|---|---|
| BR-01 | 모든 앱 기능은 인증된 사용자만 이용 가능하다. |
| BR-02 | 카테고리는 사용자별로 독립적으로 관리되며, 타 사용자와 공유되지 않는다. |
| BR-03 | 할일의 상태는 `완료`, `기한 초과`, `활성` 세 가지로 구분된다. |
| BR-04 | 기한 초과 여부는 `due_date < 오늘 날짜 AND is_completed = false` 조건으로 판단한다. |
| BR-05 | 할일은 카테고리 없이도 등록 가능하다. |
| BR-06 | 사용자는 자신이 소유한 카테고리와 할일만 조회·수정·삭제할 수 있다. |
| BR-07 | 카테고리 삭제 시, 해당 카테고리에 속한 할일은 삭제되지 않으며 '카테고리 미지정' 상태가 된다. |

## 5. 주요 유스케이스

| UC | 유스케이스 | 주요 흐름 | 성공/실패 기준 |
|---|---|---|---|
| UC-01 | 회원가입 | 이메일·비밀번호 입력 → 유효성 검사 → 계정 생성 | 성공: 신규 계정 생성 및 자동 로그인<br>실패: 중복 이메일 또는 유효하지 않은 형식 |
| UC-02 | 로그인 | 이메일·비밀번호 입력 → 인증 → 세션/토큰 발급 | 성공: 인증 토큰 반환 및 메인 페이지 이동<br>실패: 일치하는 정보 없음 |
| UC-03 | 할일 등록 | 제목·카테고리·종료일 입력 → 유효성 검사 → 저장 | 성공: 할일 목록에 즉시 반영<br>실패: 제목 누락 또는 글자 수 초과 |
| UC-04 | 할일 목록 조회 | 필터 선택 → 권한 확인 → 데이터 로드 및 표시 | 성공: 소유한 데이터만 상태별로 구분되어 표시<br>실패: 인증 만료 시 로그인 페이지 유도 |
| UC-05 | 할일 완료 처리 | 할일 선택 → 완료 상태 변경 → DB 반영 | 성공: '완료' 탭으로 이동 및 상태 업데이트<br>실패: 타인의 할일 접근 시 권한 오류 |
| UC-06 | 할일 수정/삭제 | 할일 선택 → 내용 수정/삭제 요청 → 반영 | 성공: 수정된 내용 반영 또는 목록에서 제거<br>실패: 삭제 권한 없음 또는 존재하지 않는 할일 |
| UC-07 | 카테고리 관리 | 카테고리 정보 입력 → 추가/수정/삭제 요청 | 성공: 카테고리 목록 반영 (삭제 시 소속 할일 유지)<br>실패: 동일 이름 카테고리 존재 |
