## 4. 기능 요구사항

### 4.1 사용자 정보 관리

#### FR-AUTH-01 회원가입
- 이메일과 비밀번호를 입력하여 신규 계정을 생성한다.
- 이메일은 RFC 형식을 준수해야 하며, 중복 가입을 허용하지 않는다.
- 비밀번호는 최소 8자 이상이어야 한다.
- 가입 성공 시 자동으로 로그인 처리된다.

#### FR-AUTH-02 로그인
- 이메일과 비밀번호로 인증 후 JWT 토큰을 발급한다.
- 인증 실패 시 구체적인 계정 정보 노출 없이 오류 메시지를 반환한다.

#### FR-AUTH-03 로그아웃
- 클라이언트 측에서 JWT 토큰을 삭제하여 세션을 종료한다.

#### FR-AUTH-04 회원 탈퇴
- 인증된 사용자가 계정 삭제를 요청할 수 있다.
- 탈퇴 시 해당 사용자의 모든 할일 및 카테고리 데이터가 함께 삭제된다.

---

### 4.2 할 일 관리

#### FR-TODO-01 할일 등록
- 제목(필수, 최대 50자), 카테고리(선택), 종료일(선택), 상세 내용(선택, 최대 200자)을 입력하여 할일을 생성한다.
- 등록 즉시 목록에 반영된다.

#### FR-TODO-02 할일 목록 조회
- 로그인한 사용자 본인의 할일만 조회된다.
- 할일은 `활성` / `기한 초과` / `완료` 세 가지 상태로 구분되어 표시된다.
- 카테고리 기준 필터링을 지원한다.

#### FR-TODO-03 할일 완료 처리
- 할일을 완료 상태로 변경한다.
- 완료된 할일은 `완료` 탭으로 이동한다.
- 완료 취소(되돌리기)를 지원한다.

#### FR-TODO-04 할일 수정
- 제목, 카테고리, 종료일, 상세 내용을 수정할 수 있다.
- 본인 소유의 할일만 수정 가능하다.

#### FR-TODO-05 할일 삭제
- 할일을 목록에서 영구 삭제한다.
- 본인 소유의 할일만 삭제 가능하다.

---

### 4.3 카테고리 관리

#### FR-CAT-01 카테고리 추가
- 카테고리 이름(최대 20자)을 입력하여 생성한다.
- 동일 사용자 내 중복 이름은 허용하지 않는다.

#### FR-CAT-02 카테고리 수정
- 카테고리 이름을 변경할 수 있다.
- 본인 소유의 카테고리만 수정 가능하다.

#### FR-CAT-03 카테고리 삭제
- 카테고리를 삭제하더라도 소속 할일은 삭제되지 않으며, '카테고리 미지정' 상태가 된다.
- 본인 소유의 카테고리만 삭제 가능하다.

## 5. 비기능 요구사항

### 5.1 인증 및 보안

| 항목 | 내용 |
|---|---|
| 인증 방식 | JWT (HS-512 알고리즘) |
| 토큰 저장 위치 | 클라이언트 메모리 또는 HttpOnly Cookie |
| 비밀번호 저장 | bcrypt 해시 저장 (평문 저장 금지) |
| 권한 검증 | 모든 API 요청에 대해 토큰 유효성 및 리소스 소유권 검증 |

### 5.2 플랫폼 및 UI

| 항목 | 내용 |
|---|---|
| 지원 환경 | 반응형 웹 — 모바일 (360px 이상) 및 데스크탑 (1280px 이상) |
| 디자인 시스템 | Tailwind CSS |
| 다크모드 | Phase 2 |
| 다국어 | Phase 2 |

### 5.3 가용성 및 성능

| 항목 | 내용 |
|---|---|
| API 응답 시간 | 일반 CRUD 요청 기준 평균 200ms 이하 목표 |
| 동시 사용자 | 소규모 개인 서비스 수준 (별도 SLA 미정의) |

| 프레임워크 | Express 5 |
| DB 클라이언트 | pg (Prisma 사용 금지) |

### 6.3 Database

| 항목 | 기술 |
|---|---|
| DBMS | PostgreSQL |

[Browser]

    ▼
