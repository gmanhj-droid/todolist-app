## 2. 화면 와이어프레임

### WF-01 — 로그인 페이지 (모바일, 360px)

**화면 ID**: WF-01  
**화면명 및 URL**: 로그인 페이지 (`/sign-in`)  
**설명**: 사용자가 기존 계정으로 로그인하는 진입 화면

#### ASCII 와이어프레임

```
┌────────────────────────────────────────┐
│                                        │
│          TodoList 로고                  │
│                                        │
├────────────────────────────────────────┤
│                                        │
│  로그인                                  │
│                                        │
│  ┌──────────────────────────────────┐  │
│  │ 이메일 입력                       │  │
│  │ example@test.com                 │  │
│  └──────────────────────────────────┘  │
│                                        │
│  ┌──────────────────────────────────┐  │
│  │ 비밀번호 입력                     │  │
│  │ ••••••••••••                     │  │
│  └──────────────────────────────────┘  │
│                                        │
│  ┌──────────────────────────────────┐  │
│  │        [ 로그인 ]                │  │
│  └──────────────────────────────────┘  │
│                                        │
│  ─────────────────────────────────────  │
│                                        │
│  계정이 없으신가요?                      │
│  [ 회원가입 ]                          │
│                                        │
└────────────────────────────────────────┘
```

#### UI 요소 설명

1. **Header**: 중앙 정렬 로고 (브랜드 명 "TodoList")
2. **페이지 제목**: "로그인" (회색 서브 텍스트)
3. **이메일 입력 필드**: placeholder "이메일을 입력하세요"
4. **비밀번호 입력 필드**: 마스킹 표시 (••••), placeholder "비밀번호를 입력하세요"
5. **[로그인] 버튼**: Full-width, 주색상 (파랑)
6. **회원가입 링크**: "계정이 없으신가요? [ 회원가입 ]" 링크

#### 인터랙션

- **이메일 입력**: 입력 시 리얼타임 유효성 검사 (RFC 형식)
- **비밀번호 입력**: 마스킹 처리, 보기/숨기기 아이콘 표시
- **[로그인] 버튼 클릭**:
  - 유효성 검사: 이메일, 비밀번호 필수 입력 확인
  - 성공 시: JWT 토큰 저장 → `/todos` 페이지로 리다이렉트
  - 실패 시: 토스트 메시지 표시 "이메일 또는 비밀번호가 일치하지 않습니다"
- **[회원가입] 링크 클릭**: `/sign-up` 페이지로 이동

### WF-02 — 회원가입 페이지 (모바일, 360px)

**화면 ID**: WF-02  
**화면명 및 URL**: 회원가입 페이지 (`/sign-up`)  
**설명**: 신규 사용자가 계정을 등록하는 화면

```
│                                        │
│  ┌──────────────────────────────────┐  │
│  │ 비밀번호 입력                     │  │
│  │ ••••••••••••                     │  │

│  └──────────────────────────────────┘  │
│                                        │
│  ┌──────────────────────────────────┐  │
│  │ 비밀번호 확인                     │  │

│  └──────────────────────────────────┘  │

│  └──────────────────────────────────┘  │
│                                        │
│  ─────────────────────────────────────  │
│                                        │
│  이미 계정이 있으신가요?                 │
│  [ 로그인 ]                            │
│                                        │
└────────────────────────────────────────┘
```

#### UI 요소 설명

1. **Header**: 중앙 정렬 로고
2. **페이지 제목**: "회원가입"
3. **이메일 입력 필드**: RFC 형식 유효성 검사
4. **비밀번호 입력 필드**: 마스킹, 보기/숨기기 아이콘, 힌트 텍스트 "최소 8자 이상 입력"
5. **비밀번호 확인 필드**: 비밀번호 일치 여부 실시간 표시 (체크마크 ✓ 또는 엑스 ✗)
6. **[회원가입] 버튼**: Full-width, 주색상
7. **로그인 링크**: "이미 계정이 있으신가요? [ 로그인 ]"

#### 인터랙션

- **이메일 입력**: RFC 형식 유효성 검사, 중복 가입 확인 (blur 이벤트에서 백엔드 체크)
- **비밀번호 입력**: 8자 이상 조건 실시간 표시
- **비밀번호 확인 입력**: 입력 시 비밀번호와의 일치 여부 즉시 피드백 (빨강/초록)
- **[회원가입] 버튼 클릭**:
  - 모든 필드 필수 입력 확인
  - 비밀번호 8자 이상 검증
  - 비밀번호 일치 검증
  - 성공 시: 계정 생성 → 자동 로그인 → `/todos` 리다이렉트
  - 실패 시: 오류 메시지 표시
- **[로그인] 링크 클릭**: `/sign-in` 페이지로 이동

1. **Header**:

   - 우측: 햄버거 메뉴 아이콘 [≡] (모바일 네비게이션)

2. **상태 탭**: [활성], [기한 초과], [완료] (3개 탭, 한 번에 1개 활성)
   - 기본값: "활성" 탭

3. **카테고리 필터 드롭다운**: [전체 v]
   - 옵션: 전체 + 사용자 카테고리 목록

4. **할일 카드**:
   - 체크박스 (☐ 또는 ☑)
   - 할일 제목 (최대 50자, 완료 시 취소선)
   - 카테고리 배지 (색상 표시, 선택적)
   - 종료일 표시 (우측, YYYY-MM-DD 형식)
   - 기한 초과 시 종료일 빨강 강조 표시
   - [⋮] 더보기 메뉴 (수정/삭제 옵션)

5. **FAB (+) 버튼**: 우하단 고정, 새 할일 추가 트리거

6. **빈 목록 메시지**: "활성 할일이 없습니다" 또는 해당 상태에 맞는 메시지

#### 인터랙션

- **상태 탭 클릭**: 해당 상태 할일 목록 표시 (활성/기한 초과/완료)
- **카테고리 필터 드롭다운 클릭**: 카테고리 목록 표시, 선택 시 필터 적용
- **할일 카드 체크박스 클릭**:
  - 미완료 → 완료: 해당 할일을 "완료" 탭으로 이동, 제목 취소선 표시
  - 완료 → 미완료: 다시 "활성" 탭으로 복구 (기한 지났으면 "기한 초과")
- **[⋮] 더보기 메뉴 클릭**: 팝오버 메뉴 표시 (수정 / 삭제)
  - "수정" 클릭: WF-06 할일 수정 모달 오픈
  - "삭제" 클릭: WF-07 삭제 확인 모달 오픈
- **할일 카드 텍스트 클릭**: 할일 상세 보기 또는 WF-06 수정 모달 오픈
- **FAB (+) 버튼 클릭**: WF-06 할일 추가 모달 오픈
- **햄버거 메뉴 [≡] 클릭**: 사이드 드로어 또는 바텀 시트 표시
  - "카테고리 관리" 링크
  - "설정" 링크 (향후)
  - "로그아웃" 버튼

    - 소속 할일들의 카테고리를 "카테고리 미지정" 상태로 변경
    - 토스트 메시지: "카테고리가 삭제되었습니다. 소속 할일은 카테고리 미지정 상태가 되었습니다."
    - 모달 닫기

## 8. 마이크로인터랙션 (Micro-interactions)

### 버튼 호버 및 클릭

```
호버: 배경색 10% 밝히기, 약간의 그림자 추가
클릭: 배경색 20% 어둡히기, 약간의 누르는 느낌
활성화 (disabled): 불투명도 50%, 커서 not-allowed
```

### 입력 필드

```
포커스: 테두리 색상을 Primary 색상으로 변경, 아래쪽 그림자 추가
오류: 테두리를 Error 색상(빨강)으로 표시, 오류 메시지 표시
성공: 체크마크 아이콘 표시, 테두리를 Success 색상으로 표시
```

### 할일 카드

```
호버: 그림자 증가, 배경 약간 더 밝아짐
완료 처리: 체크마크 애니메이션, 제목 취소선 페이드-인
삭제: 카드 페이드-아웃, 슬라이드-아웃
```

### 토스트 메시지

```
진입: 아래에서 위로 슬라이드-인 (200ms)
표시: 3초 유지
퇴출: 위로 슬라이드-아웃 (200ms)
```
