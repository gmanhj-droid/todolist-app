## 개요

본 문서는 개인 할일 관리 웹 애플리케이션의 주요 사용자 시나리오를 정의합니다. 두 가지 페르소나(김철수, 박소영)를 중심으로 실제 사용 맥락을 반영한 7가지 시나리오를 제시합니다.

## 페르소나 요약

### 페르소나 A — 김철수 (27세, IT 회사 신입 개발자)
- **주요 목표**: 업무 태스크와 개인 할일을 한 곳에서 카테고리별로 관리
- **Pain Point**: 기한 초과 항목을 놓치는 경우가 많음
- **사용 환경**: 출퇴근 중 모바일, 업무 중 데스크탑
- **기대치**: 빠른 등록 및 상태 추적, 데스크탑/모바일 동기화

### 페르소나 B — 박소영 (34세, 전업주부)
- **주요 목표**: 장보기, 병원 예약 등 가사 일정을 카테고리별로 정리
- **Pain Point**: 완료/미완료 항목이 섞여 혼란스럽고 메모를 잃어버림
- **사용 환경**: 스마트폰 (모바일 위주)
- **기대치**: 직관적인 UI, 명확한 상태 구분, 카테고리 기반 정렬

## 시나리오

### SC-01: 신규 사용자 회원가입 및 첫 할일 등록

#### 시나리오 ID
SC-01

#### 시나리오 제목
신규 사용자가 회원가입하고 첫 번째 할일을 등록한다.

#### 페르소나
김철수 (27세, IT 회사 신입 개발자)

#### 사전 조건
- 애플리케이션이 정상적으로 배포됨
- 사용자가 미가입 상태임
- 사용자가 데스크탑 브라우저에서 접근함

#### 단계별 행동 흐름

| 단계 | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | 회원가입 버튼 클릭 | 회원가입 양식 페이지 표시 |
| 2 | 이메일(chulsu.kim@example.com) 입력 | 입력값 표시 |
| 3 | 비밀번호(8자 이상) 입력 | 비밀번호 강도 인디케이터 표시 |
| 4 | 비밀번호 확인 입력 | 일치 여부 확인 메시지 표시 |
| 5 | "회원가입" 버튼 클릭 | 서버에 가입 정보 전송 |
| 6 | - | 계정 생성 완료, 자동 로그인 처리 |
| 7 | - | 메인 대시보드로 리다이렉트 |
| 8 | "새 할일 추가" 버튼 클릭 | 할일 생성 양식 표시 |
| 9 | 제목("프로젝트 A 요구사항 분석 완료") 입력 | 입력값 표시 |
| 10 | 기한(2026-05-05) 선택 | 달력 피커 표시, 날짜 선택 |
| 11 | "저장" 버튼 클릭 | 할일 목록에 "프로젝트 A 요구사항 분석 완료" 추가 (활성 상태) |

#### 성공 기준
- 이메일, 비밀번호로 계정이 성공적으로 생성됨
- 가입 직후 자동으로 로그인됨
- 첫 할일이 대시보드에 표시됨
- 할일이 "활성" 상태로 저장됨
- 입력한 기한이 올바르게 저장됨

#### 예외/실패 흐름

**이메일 중복 가입 시도:**
| 단계 | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | 이미 가입된 이메일로 회원가입 시도 | "이미 가입된 이메일입니다" 오류 메시지 표시 |
| 2 | 다른 이메일로 재입력 | 정상 진행 |

**비밀번호 불일치:**
| 단계 | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | 비밀번호와 확인 비밀번호가 다름 | "비밀번호가 일치하지 않습니다" 메시지 표시 |
| 2 | "회원가입" 버튼 비활성화 | - |

### SC-02: 카테고리 생성 후 할일 분류 등록

#### 시나리오 ID
SC-02

#### 시나리오 제목
업무와 개인 일정을 구분하기 위해 카테고리를 생성하고 할일을 분류하여 등록한다.

#### 페르소나
김철수 (27세, IT 회사 신입 개발자)

#### 사전 조건
- 사용자가 로그인된 상태임
- 메인 대시보드에 있음
- 기존 카테고리가 없음

#### 단계별 행동 흐름

| 단계 | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | "카테고리 관리" 메뉴 접근 | 카테고리 목록 페이지 표시 (비어있음) |
| 2 | "새 카테고리 추가" 버튼 클릭 | 카테고리 생성 양식 표시 |
| 3 | 카테고리명("업무") 입력 | 입력값 표시 |
| 4 | "생성" 버튼 클릭 | "업무" 카테고리 생성, 목록에 추가 |
| 5 | "새 카테고리 추가" 버튼 재클릭 | 카테고리 생성 양식 표시 |
| 6 | 카테고리명("개인") 입력 | 입력값 표시 |
| 7 | "생성" 버튼 클릭 | "개인" 카테고리 생성, 목록에 추가 |
| 8 | 메인 대시보드로 이동 | - |
| 9 | "새 할일 추가" 버튼 클릭 | 할일 생성 양식 표시 |
| 10 | 제목("코드 리뷰 완료") 입력 | 입력값 표시 |
| 11 | 카테고리 드롭다운에서 "업무" 선택 | "업무" 카테고리 선택됨 |
| 12 | 기한(2026-04-30) 선택 | 기한 설정 |
| 13 | "저장" 버튼 클릭 | 할일이 "업무" 카테고리에 추가됨 |
| 14 | "새 할일 추가" 버튼 재클릭 | 할일 생성 양식 표시 |
| 15 | 제목("짐 정리하기") 입력 | 입력값 표시 |
| 16 | 카테고리 드롭다운에서 "개인" 선택 | "개인" 카테고리 선택됨 |
| 17 | 기한(2026-05-10) 선택 | 기한 설정 |
| 18 | "저장" 버튼 클릭 | 할일이 "개인" 카테고리에 추가됨 |
| 19 | 대시보드 조회 | "업무" 카테고리 아래 "코드 리뷰 완료", "개인" 카테고리 아래 "짐 정리하기" 표시 |

#### 성공 기준
- 두 개의 카테고리("업무", "개인")가 성공적으로 생성됨
- 할일 생성 시 카테고리를 선택할 수 있음
- 각 할일이 선택된 카테고리에 올바르게 분류됨
- 대시보드에서 카테고리별로 할일이 그룹화되어 표시됨

#### 예외/실패 흐름

**중복 카테고리명 입력:**
| 단계 | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | "업무" 카테고리가 이미 존재한 상태에서 동일명 입력 | "이미 존재하는 카테고리명입니다" 오류 메시지 표시 |
| 2 | 다른 이름으로 재입력 | 정상 진행 |

**카테고리 미선택:**
| 단계 | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | 할일 생성 시 카테고리를 선택하지 않고 저장 버튼 클릭 | 할일이 "카테고리 미지정" 상태로 저장됨 |

### SC-03: 기한 초과 할일 확인 및 완료 처리

#### 시나리오 ID
SC-03

#### 시나리오 제목
기한이 지난 할일을 확인하고 완료 처리한다.

#### 페르소나
김철수 (27세, IT 회사 신입 개발자)

#### 사전 조건
- 사용자가 로그인된 상태임
- 다음 할일들이 등록되어 있음:
  - "프로젝트 제안서 작성" (카테고리: 업무, 기한: 2026-04-25, 상태: 활성)
  - "회의 자료 준비" (카테고리: 업무, 기한: 2026-04-27, 상태: 활성)
  - "코드 리뷰" (카테고리: 업무, 기한: 2026-05-05, 상태: 활성)
- 현재 날짜: 2026-04-28
- 데스크탑 브라우저 사용 중

#### 단계별 행동 흐름

| 단계 | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | 메인 대시보드 접근 | 할일 목록 표시: "활성" 항목 3개, "기한 초과" 항목 2개로 분류 |
| 2 | "기한 초과" 필터 또는 섹션 확인 | "프로젝트 제안서 작성"(기한 3일 초과), "회의 자료 준비"(기한 1일 초과) 표시 |
| 3 | "프로젝트 제안서 작성" 항목의 완료 버튼 클릭 | 해당 할일이 "완료" 상태로 변경 |
| 4 | 할일 목록 재조회 | "기한 초과" 섹션에 "회의 자료 준비"만 남음, "완료" 섹션에 "프로젝트 제안서 작성" 추가 |
| 5 | "회의 자료 준비" 할일 선택 | 할일 상세 페이지 표시 |
| 6 | "완료" 버튼 클릭 | 할일 상태가 "완료"로 변경 |
| 7 | 대시보드로 돌아감 | "기한 초과" 섹션이 비어있음, "완료" 섹션에 2개 항목 표시 |

#### 성공 기준
- 기한이 지난 활성 할일이 "기한 초과" 상태로 올바르게 표시됨
- 완료 처리된 할일이 "완료" 섹션으로 이동됨
- 기한 초과 날짜 정보가 사용자에게 명확하게 표시됨
- 상태 변경이 실시간으로 반영됨

#### 예외/실패 흐름

**완료 처리 후 실행 취소:**
| 단계 | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | 완료된 할일의 "완료 취소" 또는 "다시 활성화" 버튼 클릭 | 할일 상태가 "활성" 또는 "기한 초과"로 되돌려짐 |

**중복 완료 버튼 클릭:**
| 단계 | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | 완료된 할일의 완료 버튼을 다시 클릭 | 동작 무시, 이미 완료 상태임을 알림 |

### SC-04: 모바일에서 당일 할일 등록 및 완료 처리

#### 시나리오 ID
SC-04

#### 시나리오 제목
외출 중 모바일에서 당일 할일을 빠르게 등록하고 완료 처리한다.

#### 페르소나
박소영 (34살, 전업주부)

#### 사전 조건
- 사용자가 로그인된 상태임
- 스마트폰(모바일 브라우저 또는 반응형 앱)에서 접근 중
- 현재 날짜: 2026-04-28
- 기존 카테고리: "쇼핑", "병원/의료", "집안일"

#### 단계별 행동 흐름

| 단계 | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | 마트 가는 길에 앱/웹 사이트 접근 | 모바일 반응형 대시보드 로드 |
| 2 | "새 할일 추가" 버튼(플로팅 버튼 또는 상단 버튼) 클릭 | 할일 생성 모달 또는 페이지 표시 |
| 3 | 제목("우유, 계란, 당근 사기") 입력 | 입력값 표시 |
| 4 | 카테고리에서 "쇼핑" 선택 | "쇼핑" 카테고리 선택됨 |
| 5 | 기한 설정: 오늘(2026-04-28) 선택 | 기한 자동 설정 |
| 6 | "저장" 버튼 클릭 | 할일 등록 완료, 대시보드로 돌아감 |
| 7 | 마트에서 쇼핑 완료 후 "우유, 계란, 당근 사기" 항목 확인 | 할일 항목 표시 |
| 8 | 할일의 체크박스 또는 "완료" 버튼 클릭 | 할일이 "완료" 상태로 변경, 시각적 피드백(스트라이크스루 등) 표시 |
| 9 | 집에 도착 후 앱 재접근 | 완료된 할일이 "완료" 섹션에 표시됨 |

#### 성공 기준
- 모바일 화면에서 할일 등록 폼이 사용하기 쉬운 크기로 표시됨
- 기한을 "오늘"로 빠르게 설정할 수 있음
- 체크박스 또는 완료 버튼이 모바일에서 쉽게 클릭되는 크기임
- 완료 처리가 즉시 UI에 반영됨
- 모바일에서도 데이터가 정확하게 동기화됨

#### 예외/실패 흐름

**네트워크 오류 발생:**
| 단계 | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | 할일 저장 중 네트워크 연결 끊김 | "저장 실패" 알림, 로컬 임시 저장 또는 재시도 옵션 제공 |
| 2 | 사용자가 "재시도" 버튼 클릭 | 네트워크 복구 시 할일 정상 저장 |

**기한 없이 저장:**
| 단계 | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | 기한을 설정하지 않고 저장 버튼 클릭 | 할일이 "기한 미지정" 상태로 저장됨 |

### SC-05: 카테고리 삭제 시 소속 할일 처리 확인

#### 시나리오 ID
SC-05

#### 시나리오 제목
카테고리를 삭제할 때 소속 할일이 어떻게 처리되는지 확인한다.

#### 페르소나
박소영 (34살, 전업주부)

#### 사전 조건
- 사용자가 로그인된 상태임
- 다음 카테고리와 할일이 등록되어 있음:
  - 카테고리: "병원/의료"
    - 할일 1: "목 감기약 먹기" (활성)
    - 할일 2: "정기 검진 예약" (완료)
  - 카테고리: "집안일"
    - 할일 3: "거실 청소하기" (활성)

#### 단계별 행동 흐름

| 단계 | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | "카테고리 관리" 페이지 접근 | 카테고리 목록 표시: "병원/의료", "집안일" 등 |
| 2 | "병원/의료" 카테고리 옆의 "삭제" 버튼 클릭 | 확인 다이얼로그 표시: "이 카테고리에 속한 할일 2개가 있습니다. 삭제하면 해당 할일은 '카테고리 미지정' 상태가 됩니다. 계속하시겠습니까?" |
| 3 | "확인" 버튼 클릭 | "병원/의료" 카테고리 삭제 |
| 4 | 카테고리 목록 재조회 | "병원/의료" 카테고리가 목록에서 제거됨 |
| 5 | 메인 대시보드 접근 | 할일 목록 조회 |
| 6 | 할일 목록에서 확인 | "목 감기약 먹기", "정기 검진 예약" 할일들이 "카테고리 미지정" 상태로 표시됨 |
| 7 | "목 감기약 먹기" 할일 클릭 | 할일 상세 페이지 표시, 카테고리가 비어있음 |
| 8 | "집안일" 카테고리의 "삭제" 버튼 클릭 | 확인 다이얼로그 표시 |
| 9 | "취소" 버튼 클릭 | 다이얼로그 닫음, 카테고리 유지 |

#### 성공 기준
- 카테고리 삭제 시 소속 할일의 개수를 명확하게 표시함
- 사용자가 삭제 전에 영향받을 할일들을 인식할 수 있음
- 카테고리 삭제 후 소속 할일들이 "카테고리 미지정" 상태로 정확하게 유지됨
- 기존 할일의 내용(제목, 기한, 상태)은 그대로 보존됨
- 삭제 작업을 취소할 수 있음

#### 예외/실패 흐름

| 2 | 모든 카테고리 삭제 후 대시보드 접근 | 모든 할일이 "카테고리 미지정" 상태로 표시됨 |

|------|-----------|-----------|

### SC-06: 로그인 실패 및 재시도

#### 시나리오 ID
SC-06

#### 시나리오 제목
로그인 시 잘못된 자격증명으로 로그인 실패 후 재시도한다.

#### 페르소나
공통 (김철수 또는 박소영)

#### 사전 조건
- 사용자가 미로그인 상태임
- 유효한 계정이 이미 가입되어 있음 (예: chulsu.kim@example.com)
- 로그인 페이지에 접근 가능함

#### 단계별 행동 흐름

| 단계 | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | 로그인 페이지 접근 | 로그인 폼 표시 (이메일, 비밀번호 입력 필드) |
| 2 | 이메일(chulsu.kim@example.com) 입력 | 입력값 표시 |
| 3 | 비밀번호(wrongpassword123) 입력 | 입력값 표시 (마스킹됨) |
| 4 | "로그인" 버튼 클릭 | 서버에서 자격증명 검증 |
| 5 | - | "이메일 또는 비밀번호가 일치하지 않습니다" 오류 메시지 표시 |
| 6 | 비밀번호 필드의 입력값을 모두 삭제 | 입력값 초기화 |
| 7 | 정확한 비밀번호(correctpassword123) 입력 | 입력값 표시 (마스킹됨) |
| 8 | "로그인" 버튼 클릭 | 서버에서 자격증명 검증 |
| 9 | - | 로그인 성공, 메인 대시보드로 리다이렉트 |
| 10 | 대시보드 접근 | 사용자의 할일 목록 표시 |

#### 성공 기준
- 잘못된 자격증명 입력 시 명확한 오류 메시지가 표시됨
- 오류 메시지가 특정 필드의 오류를 명확히 하지 않음 (보안: 어느 필드가 잘못되었는지 공개하지 않음)
- 사용자가 오류 후 다시 입력할 수 있음
- 정확한 자격증명으로 성공적으로 로그인됨
- 로그인 성공 후 대시보드가 제대로 로드됨

#### 예외/실패 흐름

**계정 없음:**

| 2 | 비밀번호 필드를 비운 채로 "로그인" 버튼 클릭 | "비밀번호는 필수 입력 항목입니다" 유효성 검사 메시지 표시, 폼 제출 불가 |

### SC-07: 회원 탈퇴

#### 시나리오 ID
SC-07

#### 시나리오 제목
사용자가 계정을 탈퇴하고 모든 데이터가 삭제된다.

#### 페르소나
공통 (김철수 또는 박소영)

#### 사전 조건
- 사용자가 로그인된 상태임
- 사용자 계정에 다음 데이터가 있음:
  - 카테고리 3개
  - 할일 10개
  - 완료된 할일 5개

#### 단계별 행동 흐름

| 단계 | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | 설정 또는 계정 관리 페이지 접근 | 계정 관리 페이지 표시 |
| 2 | "회원 탈퇴" 또는 "계정 삭제" 옵션 찾기 | "회원 탈퇴" 버튼 또는 링크 표시 (빨간색 또는 경고 스타일) |
| 3 | "회원 탈퇴" 버튼 클릭 | 경고 다이얼로그 표시: "계정을 탈퇴하면 모든 할일, 카테고리, 데이터가 영구적으로 삭제됩니다. 이 작업은 되돌릴 수 없습니다. 계속하시겠습니까?" |
| 4 | "취소" 버튼 클릭 | 다이얼로그 닫음, 탈퇴 프로세스 중단 |
| 5 | 다시 설정 페이지에서 "회원 탈퇴" 버튼 클릭 | 경고 다이얼로그 재표시 |
| 6 | "확인" 버튼 클릭 | 비밀번호 재입력 페이지 표시 (보안 확인) |
| 7 | 비밀번호 입력 | 입력값 표시 (마스킹됨) |
| 8 | "탈퇴 진행" 버튼 클릭 | 비밀번호 검증 |
| 9 | - | 사용자 계정 및 모든 데이터 삭제 |
| 10 | - | 로그아웃 처리, 로그인 페이지로 리다이렉트 |
| 11 | 이전 이메일로 로그인 시도 | "이메일 또는 비밀번호가 일치하지 않습니다" 오류 메시지 (계정이 존재하지 않음) |

#### 성공 기준
- 회원 탈퇴 옵션이 설정 페이지에서 찾기 쉬운 위치에 있음
- 탈퇴 전에 데이터 손실에 대한 명확한 경고가 표시됨
- 탈퇴 확인 시 비밀번호 재입력으로 의도 재확인 가능
- 탈퇴 후 사용자 계정이 완전히 삭제됨
- 삭제된 계정으로는 로그인할 수 없음

#### 예외/실패 흐름

**탈퇴 중 취소:**
| 단계 | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | 경고 다이얼로그에서 "취소" 버튼 클릭 | 다이얼로그 닫음, 사용자 계정 유지 |
| 2 | 설정 페이지로 돌아감 | 사용자의 모든 데이터 그대로 유지 |

**잘못된 비밀번호 입력:**
| 단계 | 사용자 행동 | 시스템 반응 |
|------|-----------|-----------|
| 1 | 비밀번호 재입력 단계에서 잘못된 비밀번호 입력 | "비밀번호가 일치하지 않습니다" 오류 메시지 표시 |
| 2 | 사용자가 올바른 비밀번호로 재입력 | "탈퇴 진행" 버튼 활성화 |

## 참고 사항

### 상태 정의
- **활성(Active)**: 기한이 아직 지나지 않았고 완료되지 않은 할일
- **기한 초과(Overdue)**: 기한이 지났으나 완료되지 않은 할일 (due_date < 오늘 AND 미완료)
- **완료(Completed)**: 완료 처리된 할일

### 데이터 소유권 원칙
- 모든 기능은 인증된 사용자만 이용 가능
- 사용자는 자신의 데이터만 조회·수정·삭제 가능
- 카테고리는 사용자별 독립 관리, 타 사용자와 비공유

### 플랫폼 고려사항
- **데스크탑**: 프로젝트 기반 작업, 장시간 사용, 복잡한 작업
- **모바일**: 빠른 등록/완료, 외출 중 사용, 간단한 상호작용
